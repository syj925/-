# 校园墙审核功能说明

## 🎯 功能概述

校园墙系统现在支持灵活的内容审核机制，管理员可以根据需要选择不同的审核模式，确保平台内容质量。

## 🔧 审核模式

### 1. 强制人工审核模式
- **开关位置**: 系统设置 → 内容设置 → 强制人工审核
- **功能描述**: 开启后，**所有帖子和评论都必须经过人工审核才能显示**
- **适用场景**: 
  - 重要活动期间需要严格控制内容
  - 新系统上线初期需要严格把关
  - 发现大量违规内容需要加强管控

### 2. 智能审核模式
- **开关位置**: 系统设置 → 内容设置 → 智能审核模式
- **前提条件**: 强制人工审核关闭时才生效
- **功能描述**: 根据关键词智能判断内容是否需要审核
- **审核流程**:
  ```
  用户发布内容
      ↓
  检查自动拒绝关键词 → 包含 → 直接拒绝 ❌
      ↓ 不包含
  检查自动通过关键词 → 包含 → 自动通过 ✅
      ↓ 不包含
  智能审核开启？
      ├─ 是 → 提交人工审核 ⏳
      └─ 否 → 直接通过 ✅
  ```

### 3. 完全开放模式
- **设置方式**: 关闭强制人工审核 + 关闭智能审核模式
- **功能描述**: 只有包含自动拒绝关键词的内容会被拒绝，其他内容直接发布
- **适用场景**: 用户活跃度高，内容质量较好的成熟社区

## ⚙️ 关键词设置

### 自动通过关键词
- **位置**: 系统设置 → 内容设置 → 自动审核通过关键词
- **格式**: 用逗号分隔，如：`学习,教育,知识,分享`
- **作用**: 包含这些关键词的内容会自动审核通过
- **建议词汇**: 学习、教育、知识、分享、讨论、交流、活动、通知等正面词汇

### 自动拒绝关键词
- **位置**: 系统设置 → 内容设置 → 自动拒绝关键词
- **格式**: 用逗号分隔，如：`广告,推广,微信,QQ`
- **作用**: 包含这些关键词的内容会被直接拒绝，无法发布
- **建议词汇**: 广告、推广、微信、QQ、代理、加盟、刷单等违规词汇

## 📱 用户体验

### 发布成功（直接通过）
```
用户点击发布 → 前端验证通过 → 后端审核通过 → 内容立即显示
```

### 发布成功（进入审核）
```
用户点击发布 → 前端验证通过 → 后端标记为待审核 → 显示"内容已提交，等待审核"
```

### 发布失败（包含禁词）
```
用户点击发布 → 前端验证通过 → 后端检测到禁词 → 显示"内容包含禁止词汇，无法发布"
```

## 🔍 管理员操作

### 查看待审核内容
1. 登录管理后台
2. 进入 **内容管理** → **内容审核**
3. 查看待审核的帖子和评论列表

### 审核操作
- **通过审核**: 内容立即显示给所有用户
- **拒绝审核**: 内容不会显示，用户可以看到被拒绝的提示
- **批量操作**: 支持批量通过或拒绝多个内容

## 📊 状态说明

### 帖子状态
- `published`: 已发布（正常显示）
- `pending`: 待审核（不显示给普通用户）
- `rejected`: 审核被拒（不显示）
- `pinned`: 置顶帖子（优先显示）

### 评论状态
- `normal`: 正常状态（正常显示）
- `pending`: 待审核（不显示给普通用户）
- `rejected`: 审核被拒（不显示）

## ⚠️ 注意事项

### 关键词设置
1. **区分大小写**: 系统会自动忽略大小写
2. **完全匹配**: 关键词需要完全包含在内容中才会触发
3. **优先级**: 自动拒绝关键词优先级高于自动通过关键词
4. **定期更新**: 建议根据实际情况定期调整关键词列表

### 审核工作量
1. **强制审核**: 会产生大量审核工作，建议配备足够的管理员
2. **智能审核**: 可以大幅减少审核工作量，推荐使用
3. **关键词优化**: 合理设置关键词可以进一步减少人工审核量

### 用户体验
1. **及时审核**: 待审核内容应该尽快处理，避免用户等待过久
2. **审核反馈**: 被拒绝的内容应该给出明确的拒绝原因
3. **透明度**: 可以在用户协议中说明审核机制
